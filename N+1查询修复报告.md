# N+1查询性能问题修复报告

## 问题概述
**问题ID**: P1-4
**文件**: `api/app/controller/Statistics.php`
**问题类型**: N+1查询性能问题

## 问题描述
在设备统计查询中，对每个设备都执行单独的数据库查询来获取：
1. 设备触发统计（总次数和成功次数）
2. 设备最后触发时间

当有100台设备时，会产生约201次数据库查询（1次查询设备列表 + 100次查询统计 + 100次查询最后触发时间）。

## 修复方案

### 优化技术
使用 **GROUP BY 聚合查询** + **WHERE IN 批量查询**，将多个单独查询合并为2次批量查询。

### 修复位置

#### 1. deviceStats() 方法（第266-302行）
```php
// 优化前：对每个设备单独查询
foreach ($devices as $device) {
    $triggerCount = DeviceTrigger::where('device_id', $device->id)
        ->where('create_time', '>=', $startDate . ' 00:00:00')
        ->where('create_time', '<=', $endDate . ' 23:59:59')
        ->count();

    $successCount = DeviceTrigger::where('device_id', $device->id)
        ->where('create_time', '>=', $startDate . ' 00:00:00')
        ->where('create_time', '<=', $endDate . ' 23:59:59')
        ->where('success', 1)
        ->count();

    $lastTrigger = DeviceTrigger::where('device_id', $device->id)
        ->where('success', 1)
        ->order('create_time', 'desc')
        ->value('create_time');
}

// 优化后：使用GROUP BY一次性获取所有设备统计
$deviceIds = array_column($devices->toArray(), 'id');

// 一次性获取触发统计
$statsData = DeviceTrigger::whereIn('device_id', $deviceIds)
    ->where('create_time', '>=', $startDate . ' 00:00:00')
    ->where('create_time', '<=', $endDate . ' 23:59:59')
    ->field('device_id, COUNT(*) as total_count, SUM(CASE WHEN success = 1 THEN 1 ELSE 0 END) as success_count')
    ->group('device_id')
    ->select()
    ->toArray();

// 一次性获取最后触发时间

    ->where('success', 1)
    ->field('device_id, MAX(create_time) as last_trigger_time')
    ->group('device_id')
    ->select()
    ->toArray();
```

### SQL优化示例

#### 优化前
```sql
-- 每个设备执行3次查询（循环100次）
SELECT COUNT(*) FROM device_triggers WHERE device_id = 1 AND create_time >= '...';
SELECT COUNT(*) FROM device_triggers WHERE device_id = 1 AND success = 1 AND create_time >= '...';
SELECT MAX(create_time) FROM device_triggers WHERE device_id = 1 AND success = 1;
```

#### 优化后
```sql
-- 一次性获取所有设备的统计数据
SELECT device_id,
       COUNT(*) as total_count,
       SUM(CASE WHEN success = 1 THEN 1 ELSE 0 END) as success_count
FROM device_triggers
WHERE device_id IN (1,2,3,...,100) AND create_time >= '...'
GROUP BY device_id;

-- 一次性获取所有设备的最后触发时间
SELECT device_id, MAX(create_time) as last_trigger_time
FROM device_triggers
WHERE device_id IN (1,2,3,...,100) AND success = 1
GROUP BY device_id;
```

## 关键优化技术

1. **批量查询**: 使用 `WHERE IN` 一次性查询多个设备
2. **聚合查询**: 使用 `COUNT()`, `SUM()`, `MAX()` 等聚合函数
3. **GROUP BY**: 按 `device_id` 分组，一条记录返回一个设备的统计
4. **内存映射**: 将查询结果映射到数组，通过设备ID快速查找
