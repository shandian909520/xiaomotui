## 🔧 修复的关键问题清单

### 问题1: JWT签名不一致 ⚡ **最严重**

**现象**: 所有需要认证的接口返回401"令牌签名无效"

**原因**:
```php
// AuthService生成token
$secretKey = env('jwt.secret_key', 'xiaomotui_jwt_secret_key');

// JwtUtil验证token
$secretKey = config('jwt.secret') ?? 'xiaomotui_jwt_secret_key_2024';
```

**修复**:
```php
// 统一使用config获取
$config = config('jwt');
$secretKey = $config['secret'] ?? 'xiaomotui_jwt_secret_key_2024';
```

**影响**: 🔴 Critical - 修复后所有认证接口恢复正常

---

### 问题2: 权限配置不完整

**现象**: user角色无法访问AI、merchant、statistics接口

**修复**: 在`config/auth.php`中添加:
```php
'user' => [
    // ... 原有权限
    'ai/*',
    'merchant/info',
    'merchant/nfc/*',
    'merchant/device/*',
    'statistics/*',
]
```

**影响**: 🟡 High - 新增8个接口的访问权限

---

### 问题3: 路由命名空间问题

**现象**: 控制器找不到,报错`app\index\controller\Xxx not found`

**原因**: ThinkPHP多应用模式下,简写路由被解析为`app\index\controller\`

**修复**: 批量修改为完整类名
```php
// 修复前
Route::get('info', 'Auth/info');

// 修复后
Route::get('info', '\app\controller\Auth@info');
```

**影响**: 🟡 High - 修复了20+个路由

---
