## 修复的主要问题

### 1. JWT认证签名不一致 ⚠️ **关键问题**

**问题描述**: AuthService生成token时使用的密钥与JwtUtil验证时使用的密钥不一致

**修复方式**:
```php
// 修复前
$secretKey = env('jwt.secret_key', 'xiaomotui_jwt_secret_key');

// 修复后
$config = config('jwt');
$secretKey = $config['secret'] ?? 'xiaomotui_jwt_secret_key_2024';
```

**影响**: 所有需要认证的接口都无法使用

### 2. 权限配置不完整

**问题描述**: user角色缺少AI、merchant和statistics相关权限

**修复方式**: 在`config/auth.php`中为user角色添加权限:
```php
'user' => [
    // ... 其他权限
    'ai/*',
    'merchant/info',
    'merchant/nfc/*',
    'merchant/device/*',
    'statistics/*',
]
```

### 3. 路由控制器命名空间问题

**问题描述**: 需要认证的路由使用简写方式,在多应用模式下被解析为错误的命名空间

**修复方式**:
```php
// 修复前
Route::get('info', 'Auth/info');

// 修复后
Route::get('info', '\app\controller\Auth@info');
```

## 技术架构确认

### 认证流程 ✅ 正常

1. 用户请求发送验证码
2. 服务端生成验证码并缓存(5分钟有效期)
3. 用户使用验证码登录
4. 服务端生成JWT token (24小时有效期)
5. 后续请求携带Bearer token
6. Auth中间件验证token并注入用户信息

### JWT配置 ✅ 正确

- 算法: HS256
- Secret: xiaomotui_jwt_secret_key_2024
- 过期时间: 86400秒 (24小时)
- 刷新令牌: 604800秒 (7天)
- Token前缀: Bearer

- 支持3种角色: admin, merchant, user
- 基于路由模式的权限控制
- 管理员拥有所有权限 (*)
- 商家和用户有细粒度权限控制
