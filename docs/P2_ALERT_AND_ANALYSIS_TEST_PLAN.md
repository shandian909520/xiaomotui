# P2优先级功能测试计划：告警推送+数据分析

## 一、测试目标

验证告警推送和数据分析功能的完整性和稳定性，确保系统能够：

1. 实时监控设备状态并触发告警
2. 通过多渠道推送告警通知
3. 提供完整的数据分析和可视化
4. 支持WebSocket实时通信

## 二、测试环境

### 2.1 环境要求

- **后端**: PHP 8.0+, ThinkPHP 8.0, MySQL 8.0+, Redis 6.0+
- **前端**: Vue 3.0+, Element Plus, ECharts 5.0+
- **WebSocket**: WebSocket服务器(端口9501)

### 2.2 测试数据准备

```bash
# 创建测试设备
php think create:test-data --devices=10

# 创建测试告警
php think create:test-data --alerts=50

# 创建测试统计数据
php think create:test-data --statistics=30days
```

### 3.3 WebSocket实时推送测试

#### TC-005: WebSocket连接测试

| 测试项 | 测试步骤 | 预期结果 |
|--------|----------|----------|
| 连接建立 | 1. 打开前端页面<br>2. 观察Network | WebSocket连接成功 |
| 心跳保持 | 1. 连接成功<br>2. 等待30秒 | 每30秒发送ping |
| 自动重连 | 1. 断开WebSocket<br>2. 等待5秒 | 自动重新连接 |
| 多商家隔离 | 1. 商家A连接<br>2. 商家B触发告警 | A收到,B不收到 |

**测试代码**:
```javascript
// 前端WebSocket测试
const ws = new WebSocket('ws://localhost:9501')

ws.onopen = () => {
  console.log('✓ WebSocket连接成功')
  // 测试发送消息
  ws.send(JSON.stringify({ type: 'ping', data: 'test' }))
}

ws.onmessage = (event) => {
  console.log('✓ 收到消息:', event.data)
}

ws.onerror = (error) => {
  console.error('✗ WebSocket错误:', error)
}

ws.onclose = () => {
  console.log('✗ WebSocket关闭')
  // 测试自动重连
  setTimeout(() => ws.connect(), 5000)
}
```

## 六、异常测试

### 6.1 异常场景测试

| 场景 | 测试操作 | 预期处理 |
|-----|----------|----------|
| WebSocket断开 | 断开网络 | 自动重连,显示离线状态 |
| 通知渠道失败 | 关闭webhook服务 | 降级到其他渠道,记录失败日志 |
| 大量告警并发 | 100个设备同时离线 | 队列处理,不阻塞 |
| 数据分析超时 | 查询超大数据量 | 返回缓存数据或降级处理 |
| 前端长时间挂起 | 页面打开24小时 | WebSocket心跳保持,不超时 |

### 6.2 数据一致性测试

| 测试项 | 操作 | 验证 |
|--------|-----|------|

| 缓存更新 | 更新配置 | 缓存失效,重新加载 |
| 并发处理 | 多人同时处理同一告警 | 只有一个处理成功 |
