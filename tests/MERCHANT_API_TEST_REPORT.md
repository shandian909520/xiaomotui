## 一、测试概述

### 1.1 测试目标

对商家管理模块的所有功能接口进行全面测试，验证：
- 功能完整性
- 数据验证
- 权限控制
- 错误处理
- API响应性能

### 1.2 测试范围

#### 登录认证 (2个接口)
- POST `/api/auth/phone-login` - 手机号登录
- GET `/api/auth/info` - 获取用户信息

#### 商家信息管理 (3个接口)
- GET `/api/merchant/info` - 获取商家信息
- POST `/api/merchant/update` - 更新商家信息
- GET `/api/merchant/statistics` - 获取商家统计

#### NFC设备管理 (8个接口)
- GET `/api/nfc/devices` - NFC设备列表
- GET `/api/nfc/stats` - NFC设备统计
- GET `/api/nfc/trigger-records` - 获取触发记录
- GET `/api/nfc/device/:id/records` - 获取设备触发记录
- GET `/api/nfc/device/:id/stats` - 获取设备统计

#### 团购管理 (3个接口)
- PUT `/api/nfc/device/:device_id/group-buy` - 配置团购
- GET `/api/nfc/device/:device_id/group-buy` - 获取团购配置
- GET `/api/group-buy/statistics` - 团购统计

#### 商家模板管理 (4个接口)
- GET `/api/template/list` - 模板列表
- POST `/api/template/create` - 创建模板
- PUT `/api/template/:id` - 更新模板
- DELETE `/api/template/:id` - 删除模板

#### 优惠券管理 (5个接口)
- GET `/api/coupon/list` - 优惠券列表
- POST `/api/coupon/create` - 创建优惠券
- PUT `/api/coupon/:id` - 更新优惠券
- DELETE `/api/coupon/:id` - 删除优惠券
- GET `/api/coupon/:id/usage` - 优惠券使用情况

## 四、发现的问题

### 4.1 功能问题

#### 问题1: 路由配置不一致
**描述**:
- 路由配置中部分接口使用POST，部分使用PUT
- 更新类接口应该统一使用PUT方法

**影响**: 中等
**状态**: 待修复

**建议**:
```php
// 路由配置
Route::put('template/:id', 'TemplateManage/update');  // 应使用PUT
Route::put('coupon/:id', 'Merchant/updateCoupon');    // 应使用PUT
```

---

#### 问题2: 商家ID获取逻辑复杂
**描述**:
- 商家控制器中通过user_id查找merchant_id的逻辑冗余
- 应该在JWT中间件中统一处理

**影响**: 中等
**状态**: 待优化

**建议**:
```php
// 在Auth中间件中统一设置merchant_id
if ($role === 'merchant') {
    $request->merchant_id = $payload['merchant_id'];
}
```

- 设备列表（TTL: 5分钟）
- 统计数据（TTL: 10分钟）

**实现示例**:
```php
public function getCachedMerchantInfo($merchantId) {
    $cacheKey = "merchant:info:{$merchantId}";

    $info = Cache::get($cacheKey);
    if ($info) {
        return $info;
    }

    $info = Merchant::find($merchantId);
    Cache::set($cacheKey, $info, 3600);

    return $info;
}
```
