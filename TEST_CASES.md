## 1. 核心链路测试用例

### 1.1 NFC 触发测试

#### TC-NFC-001：用户手机碰一碰 NFC 设备触发内容

**用例编号：** TC-NFC-001
**测试场景：** 用户使用手机 NFC 功能触碰设备，系统返回配置的触发内容
**前置条件：**
- NFC 设备已在系统中注册（device_id 存在）
- 设备状态为启用（status = 1）
- 设备已绑定商家（merchant_id 不为空）
- 设备已配置触发内容（trigger_type 和 trigger_content 已设置）

**测试步骤：**
1. 准备测试数据：device_id = "TEST_DEVICE_001"
2. 发送 POST 请求到 `/api/nfc/trigger`
3. 请求体包含：
   ```json
   {
     "device_id": "TEST_DEVICE_001",
     "user_id": 1001,
     "trigger_time": "2026-02-12 10:30:00"
   }
   ```
4. 检查响应状态码
5. 验证返回数据结构

**预期结果：**
- HTTP 状态码：200
- 响应格式：
  ```json
  {
    "code": 200,
    "message": "触发成功",
    "data": {
      "trigger_id": 12345,
      "trigger_type": "coupon",
      "trigger_content": {
        "coupon_id": 100,
        "coupon_name": "新用户优惠券",
        "discount": "8折"
      },
      "device_info": {
        "device_name": "门店001-收银台",
        "merchant_name": "测试咖啡店"
      }
    }
  }
  ```
- 数据库 `device_triggers` 表新增一条记录
