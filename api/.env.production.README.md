## 基础配置

### 应用配置

```ini
APP_DEBUG = false
APP_TRACE = false
```

- **APP_DEBUG**: 生产环境必须设置为 `false`，关闭调试模式
- **APP_TRACE**: 生产环境必须设置为 `false`，关闭请求追踪
- **DEFAULT_TIMEZONE**: 设置为 `Asia/Shanghai`，使用中国时区

## 数据库配置

### 主库配置（写操作）

```ini
[DATABASE]
HOSTNAME = YOUR_PRODUCTION_DB_HOST      # 生产数据库主库地址
DATABASE = xiaomotui_prod               # 数据库名称
USERNAME = YOUR_DB_USERNAME             # 数据库用户名
PASSWORD = YOUR_DB_PASSWORD             # 数据库密码
HOSTPORT = 3306                         # 端口
```

**配置说明：**
- 主库用于处理所有写操作（INSERT、UPDATE、DELETE）
- 建议使用独立的数据库账号，仅授予必要的权限
- 密码必须使用强密码（建议16位以上，包含大小写字母、数字和特殊字符）

### 从库配置（读操作）

```ini
SLAVE.HOSTNAME = YOUR_SLAVE_DB_HOST     # 从库地址
SLAVE.DATABASE = xiaomotui_prod         # 数据库名称（与主库相同）
SLAVE.USERNAME = YOUR_SLAVE_DB_USERNAME # 从库用户名
SLAVE.PASSWORD = YOUR_SLAVE_DB_PASSWORD # 从库密码
```

**读写分离配置：**
```ini
DEPLOY = 1              # 启用读写分离
RW_SEPARATE = true      # 开启读写分离
MASTER_NUM = 1          # 主库数量
READ_MASTER = false     # 是否从主库读取
```

### 数据库连接池配置

```ini
POOL.MIN_CONNECTIONS = 10      # 最小连接数
POOL.MAX_CONNECTIONS = 50      # 最大连接数
POOL.CONNECT_TIMEOUT = 10.0    # 连接超时（秒）
POOL.WAIT_TIMEOUT = 3.0        # 等待超时（秒）
POOL.HEARTBEAT = 60            # 心跳间隔（秒）
POOL.MAX_IDLE_TIME = 300       # 最大空闲时间（秒）
```

**优化建议：**
- 根据服务器性能和并发量调整连接池大小
- 推荐配置：小型项目（10-20）、中型项目（20-50）、大型项目（50-100）

## Redis配置

### 基础配置

```ini
[REDIS]
HOST = YOUR_REDIS_HOST          # Redis服务器地址
PORT = 6379                     # Redis端口
PASSWORD = YOUR_REDIS_PASSWORD  # Redis密码
SELECT = 0                      # 数据库索引
PREFIX = xmt:prod:              # 键前缀（区分环境）
```

**安全建议：**
- 生产环境必须设置强密码
- 建议使用防火墙限制访问IP
- 定期更新Redis版本

### Redis连接池配置

```ini
POOL.MIN_CONNECTIONS = 10      # 最小连接数
POOL.MAX_CONNECTIONS = 30      # 最大连接数
POOL.CONNECT_TIMEOUT = 10.0    # 连接超时
POOL.WAIT_TIMEOUT = 5.0        # 等待超时
```

### Redis集群配置（可选）

如果使用Redis集群，取消以下配置的注释并填写：

```ini
# CLUSTER.NODE1 = YOUR_REDIS_CLUSTER_NODE1:6379
# CLUSTER.NODE2 = YOUR_REDIS_CLUSTER_NODE2:6379
# CLUSTER.NODE3 = YOUR_REDIS_CLUSTER_NODE3:6379
# CLUSTER.PASSWORD = YOUR_REDIS_CLUSTER_PASSWORD
```

## 安全配置

### JWT配置

```ini
[JWT]
SECRET_KEY = YOUR_PRODUCTION_JWT_SECRET_KEY_REPLACE_WITH_STRONG_KEY
EXPIRE_TIME = 86400              # Token有效期（24小时）
REFRESH_EXPIRE_TIME = 604800     # 刷新Token有效期（7天）
ALGORITHM = HS256                # 加密算法
```

**安全要求：**
- JWT密钥必须使用随机生成的强密钥（建议64位以上）
- 可以使用以下命令生成：`openssl rand -base64 64`
- 生产环境和测试环境必须使用不同的密钥

### 加密Salt

```ini
[SECURITY]
ENCRYPT_SALT = YOUR_PRODUCTION_ENCRYPT_SALT_REPLACE_WITH_STRONG_VALUE
```

用于数据加密的盐值，必须使用强随机字符串。

### 登录安全

```ini
LOGIN_FAIL_LIMIT = 5      # 登录失败次数限制
LOGIN_LOCK_TIME = 1800    # 登录失败锁定时间（秒）
```

### 跨域配置

```ini
[CORS]
ALLOWED_ORIGINS = https://h5.xiaomotui.com,https://admin.xiaomotui.com
ALLOWED_METHODS = GET,POST,PUT,DELETE,OPTIONS
ALLOWED_HEADERS = *
MAX_AGE = 3600
```

**配置说明：**
- `ALLOWED_ORIGINS`: 允许跨域的域名列表，多个域名用逗号分隔
- 生产环境建议明确指定允许的域名，不要使用 `*`

## 云服务配置

### 微信小程序配置

```ini
[WECHAT]
APP_ID = YOUR_WECHAT_APP_ID
APP_SECRET = YOUR_WECHAT_APP_SECRET
MINI_APP_ID = YOUR_WECHAT_MINI_APP_ID
MINI_APP_SECRET = YOUR_WECHAT_MINI_APP_SECRET
```

**获取方式：**
1. 登录微信公众平台：https://mp.weixin.qq.com
2. 进入"开发" -> "开发管理" -> "开发设置"
3. 复制AppID和AppSecret

### 支付配置

```ini
[PAYMENT]
WECHAT_MCH_ID = YOUR_WECHAT_MCH_ID
WECHAT_MCH_KEY = YOUR_WECHAT_MCH_KEY
WECHAT_NOTIFY_URL = https://api.xiaomotui.com/api/wechat/pay/notify
WECHAT_CERT_PATH = /data/cert/wechat/apiclient_cert.pem
WECHAT_KEY_PATH = /data/cert/wechat/apiclient_key.pem
```

**证书配置：**
- 从微信商户平台下载API证书
- 将证书上传到服务器指定路径
- 确保证书文件权限为600（仅所有者可读写）

### 对象存储配置（阿里云OSS）

```ini
[OSS]
ACCESS_ID = YOUR_ALIYUN_OSS_ACCESS_KEY_ID
ACCESS_SECRET = YOUR_ALIYUN_OSS_ACCESS_KEY_SECRET
BUCKET = YOUR_OSS_BUCKET_NAME
ENDPOINT = oss-cn-hangzhou.aliyuncs.com
URL = https://your-bucket.oss-cn-hangzhou.aliyuncs.com
PREFIX = xiaomotui/prod/
CDN_URL = https://cdn.yourdomain.com
```

**配置步骤：**
1. 登录阿里云控制台
2. 进入"访问控制" -> "用户管理"
3. 创建用户并授予OSS权限
4. 获取AccessKey ID和AccessKey Secret
5. 如使用CDN，配置CDN加速域名

## 日志和监控配置

### 日志配置

```ini
[LOG]
CHANNEL = daily              # 日志渠道（按天存储）
LEVEL = error                # 日志级别（生产环境使用error）
PATH = /data/logs/xiaomotui/ # 日志路径
MAX_FILE_SIZE = 100          # 单文件最大大小（MB）
MAX_FILES = 30               # 日志保留天数
FORMAT = json                # 日志格式
```

**日志级别说明：**
- `debug`: 调试信息（开发环境）
- `info`: 一般信息（测试环境）
- `warning`: 警告信息
- `error`: 错误信息（生产环境推荐）

### 错误报告配置（Sentry）

```ini
[ERROR]
REPORT_ENABLED = true
REPORT_TYPE = sentry
SENTRY_DSN = YOUR_SENTRY_DSN_URL
SENTRY_ENVIRONMENT = production
SENTRY_TRACES_SAMPLE_RATE = 0.1
```

**配置步骤：**
1. 注册Sentry账号：https://sentry.io
2. 创建新项目
3. 获取DSN地址并填入配置

### 监控配置

```ini
[MONITOR]
HEALTH_CHECK.ENABLED = true
DATABASE.ENABLED = true
REDIS.ENABLED = true
ALERTS.ENABLED = true
```

**告警配置：**
```ini
ALERTS.EMAIL.ENABLED = true
ALERTS.EMAIL.RECIPIENTS = ops@yourdomain.com,dev@yourdomain.com
ALERTS.WECHAT.ENABLED = true
ALERTS.WECHAT.WEBHOOK_URL = YOUR_WECHAT_WEBHOOK_URL
ALERTS.SMS.ENABLED = true
```

## 部署流程

### 1. 准备工作

- [ ] 准备生产服务器（Linux推荐）
- [ ] 安装PHP 8.0+、MySQL 5.7+、Redis 5.0+
- [ ] 安装Nginx/Apache
- [ ] 配置SSL证书

### 2. 配置文件准备

```bash
# 复制配置模板
cd /path/to/xiaomotui/api
cp .env.production.example .env.production

# 编辑配置文件
vim .env.production
```

### 3. 填写配置项

按照本文档说明，依次填写以下配置：

1. **数据库配置**
   - 主库地址、用户名、密码
   - 从库配置（如有）

2. **Redis配置**
   - Redis地址和密码

3. **安全配置**
   - JWT密钥
   - 加密Salt

4. **云服务配置**
   - 微信配置
   - OSS配置
   - AI服务配置
   - 短信服务配置

5. **域名配置**
   - API域名
   - CDN域名

6. **监控配置**
   - 日志路径
   - 错误报告
   - 告警方式

### 4. 验证配置

```bash
# 测试数据库连接
php think test:db

# 测试Redis连接
php think test:redis

# 测试微信配置
php think test:wechat

# 测试OSS配置
php think test:oss
```

### 5. 数据库迁移

```bash
# 运行数据库迁移
php think migrate:run

# 创建初始管理员账号
php think admin:create
```

### 6. 设置文件权限

```bash
# 设置运行时目录权限
chmod -R 777 runtime/
chmod -R 777 public/uploads/

# 设置配置文件权限（仅所有者可读写）
chmod 600 .env.production
```

### 7. 配置Web服务器

#### Nginx配置示例

```nginx
server {
    listen 80;
    listen 443 ssl http2;
    server_name api.xiaomotui.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    root /path/to/xiaomotui/api/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

### 8. 启动服务

```bash
# 重启Nginx
systemctl restart nginx

# 重启PHP-FPM
systemctl restart php-fpm

# 启动队列服务
php think queue:work --daemon
```

### 9. 测试验证

```bash
# 测试API接口
curl https://api.xiaomotui.com/api/health

# 查看日志
tail -f /data/logs/xiaomotui/app.log
```

## 常见问题

### Q1: 数据库连接失败

**可能原因：**
- 数据库地址或端口错误
- 用户名或密码错误
- 防火墙未开放3306端口
- 数据库未授权远程访问

**解决方法：**
```bash
# 测试数据库连接
mysql -h YOUR_DB_HOST -u YOUR_DB_USER -p
```

### Q2: Redis连接失败

**可能原因：**
- Redis地址或端口错误
- Redis密码错误
- Redis未启动
- 防火墙未开放6379端口

**解决方法：**
```bash
# 测试Redis连接
redis-cli -h YOUR_REDIS_HOST -a YOUR_REDIS_PASSWORD ping
```

### Q3: OSS上传失败

**可能原因：**
- AccessKey错误
- Bucket名称错误
- 权限不足
- 跨域配置问题

**解决方法：**
- 检查OSS控制台配置
- 确认Bucket权限设置
- 配置跨域规则（CORS）

### Q4: 微信登录失败

**可能原因：**
- AppID或AppSecret错误
- 服务器IP未加入白名单
- 回调域名未配置

**解决方法：**
- 在微信公众平台检查配置
- 添加服务器IP到白名单
- 配置正确的业务域名
