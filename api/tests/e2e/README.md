# 端到端测试 (End-to-End Tests)

## 概述

端到端测试用于验证完整的用户流程，从NFC触发到内容生成和发布，确保系统各组件协同工作正常。

## 测试场景

### 场景1: 完整NFC到发布流程

测试完整的用户流程：

1. **用户登录** - 用户通过微信登录系统
2. **NFC设备触发** - 用户触碰NFC设备
3. **内容生成** - 系统生成内容（视频/菜单等）
4. **平台发布** - 内容发布到各平台
5. **数据验证** - 验证数据一致性

**验证点**:
- NFC响应时间 < 1秒
- 内容生成时间 < 30秒
- 数据关联正确
- 无数据丢失

### 场景2: 多设备并发流程

测试系统的并发处理能力：

1. 多个用户同时触发不同设备
2. 验证所有任务正确创建
3. 检查数据完整性
4. 验证统计准确性

**验证点**:
- 无重复触发ID
- 无数据冲突
- 无数据损坏
- 统计数据准确

### 场景3: 错误处理流程

测试系统的错误处理能力：

1. 触发不存在的设备
2. 使用无效的用户
3. 内容生成失败
4. 验证错误日志

**验证点**:
- 正确返回错误响应
- 错误信息准确
- 错误日志完整
- 无孤儿记录

### 场景4: 完整商家工作流

测试商家相关的完整流程：

1. 商家注册验证
2. 设备绑定验证
3. 模板配置验证
4. NFC触发
5. 统计数据验证

**验证点**:
- 商家数据完整
- 设备绑定正确
- 模板应用正确
- 统计数据准确

## 数据一致性检查

测试会执行以下数据一致性检查：

1. **外键完整性** - 所有外键引用有效
2. **数据关联** - 相关记录正确关联
3. **孤儿记录** - 无孤儿记录存在
4. **统计准确性** - 统计数据与实际一致
5. **时间戳一致性** - 时间戳逻辑正确

## 快速开始

### 前置要求

- PHP 8.0+
- MySQL 数据库
- Composer 依赖已安装

### 运行测试

**Windows:**
```bash
cd D:\xiaomotui\api\tests\e2e
run_e2e.bat
```

**Linux/Mac:**
```bash
cd /path/to/xiaomotui/api/tests/e2e
chmod +x run_e2e.sh
./run_e2e.sh
```

**直接使用PHP:**
```bash
cd D:\xiaomotui\api\tests\e2e
php full_flow.php
```

### 配置选项

编辑 `config.php` 文件以自定义测试配置：

```php
return [
    // 基础配置
    'base_url' => 'http://localhost',
    'environment' => 'testing',

    // 超时配置（毫秒）
    'timeouts' => [
        'nfc_trigger' => 1000,        // NFC触发响应时间
        'content_generation' => 35000, // 内容生成时间
        'platform_publish' => 5000,    // 平台发布时间
    ],

    // 并发测试配置
    'concurrency' => [
        'enabled' => true,
        'concurrent_users' => 10,      // 并发用户数
        'concurrent_devices' => 10,    // 并发设备数
    ],

    // 数据清理配置
    'cleanup' => [
        'enabled' => true,             // 测试后清理数据
        'keep_logs' => true,           // 保留日志
    ],
];
```

## 测试数据

测试会自动创建以下测试数据：

- **测试用户**: 3个测试用户
- **测试商家**: 1个测试商家
- **测试设备**: 3个NFC测试设备
- **测试模板**: 1个视频模板
- **测试优惠券**: 1个优惠券

所有测试数据带有特殊前缀（如 `E2E_TEST_`、`test_e2e_`），方便识别和清理。

## 数据清理

测试完成后，默认会自动清理所有测试数据。可以通过配置关闭：

```php
'cleanup' => [
    'enabled' => false, // 关闭自动清理
]
```

## 故障排查

### 常见问题

1. **数据库连接失败**
   - 检查数据库配置
   - 确保数据库服务运行中
   - 验证用户名和密码

2. **超时错误**
   - 调整配置文件中的超时设置
   - 检查网络连接
   - 验证服务器性能

3. **数据一致性失败**
   - 检查数据库结构
   - 验证外键约束
   - 查看详细错误信息

4. **权限错误**
   - 确保reports目录可写
   - 检查数据库用户权限

### 调试模式

启用调试模式以获取更多信息：

```php
'debug' => true,
```

## 最佳实践

1. **在测试环境运行** - 不要在生产环境执行测试
2. **定期运行** - 建议在每次重大更新后运行
3. **查看报告** - 仔细分析测试报告
4. **修复失败** - 及时修复失败的测试
5. **保持更新** - 随功能更新测试用例

## 扩展测试

要添加新的测试场景，可以：

1. 在 `E2ETestRunner.php` 中添加新方法
2. 在 `full_flow.php` 中调用新方法
3. 更新本文档说明新场景

示例：

```php
// 在 E2ETestRunner.php 中添加
public function testNewScenario(): void
{
    echo "执行新场景: XXX流程\n";
    // 测试逻辑
}

// 在 full_flow.php 中调用
$runner->testNewScenario();
```
