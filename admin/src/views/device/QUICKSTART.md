# 设备管理页面 - 快速开始

## 访问页面

1. 启动管理后台
2. 登录系统
3. 点击左侧菜单"设备管理"
4. 或直接访问: `http://localhost:3000/devices`

## 5分钟快速上手

### 1. 查看设备列表（10秒）
- 页面会自动加载所有设备
- 每页默认显示20条设备
- 可以看到设备的状态、电量、位置等信息

### 2. 搜索设备（20秒）
```
1. 在"设备名称"输入框输入关键词，如"大厅"
2. 点击"搜索"按钮
3. 查看筛选结果
4. 点击"重置"恢复全部列表
```

### 3. 添加新设备（1分钟）
```
1. 点击"添加设备"按钮
2. 填写设备信息：
   - 设备编码: NFC009（必填，唯一）
   - 设备名称: 测试设备（必填）
   - 设备类型: 选择"桌台"（必填）
   - 设备位置: 三楼测试区（必填）
   - 设备描述: 用于测试（可选）
3. 点击"确定"按钮
4. 看到"添加成功"提示
```

### 4. 配置设备触发模式（1分钟）
```
1. 找到刚添加的设备
2. 点击"配置"按钮
3. 选择触发模式，如"视频生成"
4. 选择内容模板（如果是视频模式）
5. 设置设备状态为"在线"
6. 点击"保存"按钮
```

### 5. 编辑设备（30秒）
```
1. 点击设备行的"编辑"按钮
2. 修改设备名称或其他信息
3. 点击"确定"保存
```

### 6. 批量操作（30秒）
```
1. 勾选多个设备（点击列表左侧复选框）
2. 点击"批量启用"或"批量禁用"
3. 确认操作
```

### 7. 删除设备（20秒）
```
1. 点击设备行的"删除"按钮
2. 在确认对话框中点击"确定"
3. 设备被删除
```

## 常见场景

### 场景1：批量添加桌台设备
```
1. 准备好设备清单（编码、名称、位置）
2. 逐个点击"添加设备"
3. 填写信息并保存
4. 重复操作
```
**建议**: 设备编码使用统一规则，如 TABLE_01, TABLE_02...

### 场景2：更换设备触发模式
```
1. 搜索要更改的设备
2. 点击"配置"
3. 更改触发模式
4. 根据新模式填写配置
5. 保存
```
**注意**: 配置修改后立即生效

### 场景3：设备日常维护
```
1. 查看电量<20%的设备（红色进度条）
2. 更换电池
3. 点击"配置"将状态改为"在线"
```

### 场景4：设备故障处理
```
1. 找到故障设备
2. 点击"配置"
3. 将状态改为"维护"
4. 修复后改回"在线"
```

### 场景5：批量停用设备
```
1. 按类型筛选要停用的设备
2. 全选或多选设备
3. 点击"批量禁用"
4. 确认操作
```

## 数据说明

### 设备状态颜色
- **绿色"在线"**: 设备正常工作
- **红色"离线"**: 设备不可用
- **橙色"维护"**: 设备维护中

### 电量颜色
- **绿色**: 电量充足（>50%）
- **橙色**: 电量偏低（20-50%）
- **红色**: 电量不足（<20%）

### 触发模式
- **视频生成**: 用户触碰后生成个性化视频
- **优惠券**: 跳转到优惠券领取页面
- **WiFi连接**: 自动连接店铺WiFi
- **好友添加**: 添加企业微信
- **菜单展示**: 展示电子菜单

## 最佳实践

1. **设备编码规范**
   - 使用有意义的前缀，如 TABLE_、WALL_、COUNTER_
   - 统一长度和格式
   - 便于识别和管理

2. **设备命名规范**
   - 包含位置信息，如"一楼大厅桌台1号"
   - 简洁明了
   - 便于搜索

3. **定期检查**
   - 每天检查设备状态
   - 关注低电量设备
   - 及时处理离线设备

4. **配置管理**
   - 根据营业时间调整设备状态
   - 定期更新触发内容
   - 测试新配置

5. **批量操作**
   - 营业前批量启用
   - 打烊后批量禁用
   - 节省操作时间

## 故障排查

### 问题1: 设备列表加载失败
**解决方案**:
1. 检查网络连接
2. 刷新页面
3. 检查登录状态
4. 联系管理员

### 问题2: 无法添加设备
**可能原因**:
- 设备编码已存在
- 必填字段未填写
- 权限不足

### 问题3: 配置保存失败
**解决方案**:
1. 检查所有必填字段
2. 验证URL格式（如果有）
3. 重试操作

### 问题4: 设备状态不更新
**解决方案**:
1. 点击"刷新"按钮
2. 等待自动刷新（30秒）
3. 检查设备是否在线
